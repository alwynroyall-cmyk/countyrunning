# WRRL League AI 6.2.1 Release Notes

Date: 2026-04-03

## Summary
Release 6.2.1 introduces the new structured season layout, strict autopilot provenance flow, archive behavior for raw race files, friendlier autopilot UX, and updated settings/setup workflow.

## Inputs: Structured Layout
Season input folders are now standardised under:

- inputs/raw_data
- inputs/series
- inputs/control
- inputs/audited
- inputs/raw_data_archive

Rules:

- raw_data is the source of truth for race files and imports.
- audited files are generated by audit routines and used for scoring.
- edits are restricted to raw_data files.
- raw_data_archive is write-once by filename.

## Outputs: Structured Layout
Season output folders are now standardised under:

- outputs/publish/docx
- outputs/publish/pdf
- outputs/publish/xlsx
- outputs/audit/workbooks
- outputs/audit/manual-changes
- outputs/quality/staged-checks
- outputs/quality/data-quality
- outputs/autopilot/runs

## Autopilot Changes

- Autopilot regenerates audited files from raw_data before audit/scoring stages.
- Existing audited files are not trusted as source when raw_data is missing.
- Preflight diagnostics are printed for structured inputs.
- Failure runs write autopilot report artifacts so users can review messages.
- Raw source files are archived into raw_data_archive during audited generation.

## GUI Changes

- Dashboard includes autopilot-first flow and report viewing.
- Settings includes Set Up New Season and data path visibility.
- Data freshness indicator reflects raw_data vs audited currency.
- Autopilot completion dialog is user-friendly and includes Review Messages.

## Naming and Branding

- Product naming updated to WRRL League AI.
- Version is now 6.2.1.

## Operational Notes

- PDF conversion still depends on Microsoft Word via docx2pdf.
- For best results, configure data root in Settings and place race files in inputs/raw_data.
